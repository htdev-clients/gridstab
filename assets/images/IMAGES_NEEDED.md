# Images Needed for GridStab.com

This file lists all images that need to be sourced or created for the website.

## Required Images

### 1. Hero Background (`hero-background.jpg`)
- **Type**: Photograph
- **Subject**: Power grid, electrical infrastructure, or renewable energy
- **Dimensions**: 1920x1080px minimum
- **Format**: JPG (optimized, < 500KB)
- **Suggested Sources**:
  - Unsplash.com (search: "power grid", "electrical substation", "wind turbines", "solar panels")
  - Pexels.com (search: "electricity", "transmission lines", "renewable energy")
  - Custom photography if available
- **Style**: Professional, high-contrast, suitable for text overlay

### 2. Profile Photo (`profile-photo.jpg`)
- **Type**: Professional headshot
- **Subject**: Gilles Chaspierre
- **Dimensions**: 800x800px (square format)
- **Format**: JPG (optimized, < 150KB)
- **Style**:
  - Professional business attire
  - Well-lit, neutral background
  - Head and shoulders
  - Friendly, approachable expression

### 3. Favicon (`favicon.ico`)
- **Type**: Icon
- **Subject**: GridStab logo or "G" monogram
- **Dimensions**: Multi-size (16x16, 32x32, 64x64)
- **Format**: .ico file
- **Tools**:
  - Favicon.io for generation
  - Can be created from logo.svg
- **Style**: Simple, recognizable at small sizes

### 4. Logo (`logo.svg`)
- **Type**: Vector graphic
- **Subject**: GridStab wordmark or logomark
- **Format**: SVG
- **Colors**:
  - Primary version: Orange (#FF6719) + Dark (#1a1a1a)
  - White version for dark backgrounds
- **Style**: Modern, technical, professional
- **Note**: Currently using text-based logo in header

## Image Optimization Checklist

Before adding images to the site:
- [ ] Compress with TinyPNG or ImageOptim
- [ ] Verify dimensions match requirements
- [ ] Test load times (target: < 2 seconds on 3G)
- [ ] Add descriptive alt text
- [ ] Consider WebP format with JPG fallback for better compression

## Recommended Workflow

1. Source or create images according to specs above
2. Optimize files for web
3. Place in `/assets/images/` directory
4. Update alt text in HTML templates if needed
5. Test on local Jekyll server
6. Commit and deploy
